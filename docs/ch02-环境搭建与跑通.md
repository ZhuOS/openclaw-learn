# Ch02 · 环境搭建与跑通

> 动手之前只看架构是假的。这一章保证你从 **零** 开始能跑起来 Gateway，并在浏览器里看到 Control UI。

## 2.1 环境要求

| 项目 | 要求 | 备注 |
|------|------|------|
| Node.js | **24 推荐** / 22.14+ LTS | 低于 22.14 一定有问题 |
| 包管理器 | **pnpm 9+** 推荐 | npm / bun 也行，但源码贡献用 pnpm |
| 操作系统 | macOS / Linux / Windows | macOS 有原生 launchd 支持 |
| LLM API Key | 至少 1 个 | Anthropic / OpenAI / Google 三选一 |
| 端口 | 18789 默认 | 可改 |

**检查命令**：

```bash
node --version   # 应 ≥ v22.14 或 v24.x
pnpm --version   # 应 ≥ 9.x
```

如果 node 版本不够，用 `nvm` / `fnm` / `volta` 升级。

## 2.2 两种安装方式

### 方式 A：作为用户安装（快速体验）

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
openclaw dashboard
```

浏览器会打开 `http://127.0.0.1:18789/`，即 Control UI。

**适用**：你只想用、不改源码。

### 方式 B：从源码跑（本教程的主路径）

```bash
# 1. 克隆
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 2. 装依赖（pnpm workspace 会并行装）
pnpm install

# 3. 首次初始化：写入 ~/.openclaw/openclaw.json 骨架
pnpm openclaw setup

# 4. 构建 Control UI（产物进 dist/control-ui）
pnpm ui:build

# 5. 启动 Gateway（开发模式，文件改动自动重载）
pnpm gateway:watch
```

看到类似这样的输出就说明起来了：

```
[gateway] listening on http://127.0.0.1:18789
[gateway] protocol: HTTP(OpenAI-compat) + WS + /voiceclaw
[gateway] agents: main (default)
```

## 2.3 最小可用配置（openclaw.json）

Gateway 起来前先配一个能跑的 LLM。编辑 `~/.openclaw/openclaw.json`：

```json
{
  "agent": {
    "model": "anthropic/claude-sonnet-4-6"
  },
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-xxxxx"
    }
  },
  "gateway": {
    "port": 18789,
    "bind": "loopback"
  }
}
```

> 💡 配置解析优先级：`CLI 参数 → 环境变量 OPENCLAW_* → 配置文件 → 默认值`

**关键字段含义**：

| 字段 | 作用 |
|------|------|
| `agent.model` | 默认 Agent 使用的模型 ID（格式 `provider/model`） |
| `providers.*.apiKey` | 对应 provider 的凭证 |
| `gateway.port` | Gateway 监听端口 |
| `gateway.bind` | `loopback`（仅本机）/ `all`（网络）/ IP |
| `gateway.reload.mode` | `off / hot / restart / hybrid（默认）` |

## 2.4 验证 4 件套

起来之后，跑这 4 个命令挨个验证：

### ① 健康检查

```bash
curl http://127.0.0.1:18789/v1/models
```

应返回包含 `openclaw`、`openclaw/default`、`openclaw/main` 的 JSON。

### ② OpenAI 兼容接口

```bash
curl -X POST http://127.0.0.1:18789/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openclaw/default",
    "messages": [{"role":"user","content":"你好，介绍一下你自己"}]
  }'
```

如果 API Key 配对了，这里会实际调 LLM 并返回结果。

### ③ 诊断工具

```bash
pnpm openclaw doctor
```

它会扫描：端口占用、配置合法性、服务是否注册、各渠道登录状态。**非常好的调试起点**——后续你调任何问题，先跑一次 doctor。

### ④ 看 Agent 列表

```bash
pnpm openclaw agents list --bindings
```

默认只有一个 `main`，没配任何渠道绑定。

## 2.5 开发循环（gateway:watch 在做什么）

这是你源码学习期间**99% 时间都会用到的命令**：

```bash
pnpm gateway:watch
```

它内部做了什么：

```
tsx src/... (热编译 TypeScript)
  └── 监听文件改动
       └── hybrid reload：
             ├── 如果改动是 hot 安全项（e.g. 提示词、配置） → 内存热更
             └── 如果改动是结构性（e.g. 路由、协议） → 重启进程
```

**⚠️ 常见坑**：

- **改了 `openclaw.json` 没生效**：看日志有没有 `[config] reloaded`；hybrid 模式下部分字段必须重启，手动 `pnpm openclaw gateway restart`。
- **端口被占用**：先 `lsof -i:18789` 看看是不是之前的 daemon 还在，`openclaw gateway stop` 干掉。
- **pnpm install 慢**：WhatsApp 的 Baileys 和 Puppeteer 体积很大；可以 `PUPPETEER_SKIP_DOWNLOAD=1 pnpm install` 跳过。

## 2.6 Control UI 是什么

`http://127.0.0.1:18789/` 打开的是 **Control UI**，一个基于 React + Vite 的 SPA，源码在 `ui/` 目录。它的作用：

- 看会话（哪个 Agent、哪个渠道、哪些消息）
- 看 Health / Presence 面板
- 管理渠道登录（扫 WhatsApp 二维码等）
- 改配置（实时 patch 到 `openclaw.json`）

它和 Gateway 是**同源部署**：Vite 构建产物放进 `dist/control-ui/`，Gateway 的 HTTP server 直接 serve 这个目录。没有独立后端。

读 UI 代码时你只需要关注：它通过 **WS 协议** 和 Gateway 通信，协议细节我们在 Ch03 拆。

## 2.7 连第一个渠道：Telegram（最快）

为了后面章节有实际流量可观察，**先把 Telegram 接上**，它是最简单的：

```bash
# 1. 在 Telegram 上找 @BotFather，/newbot 拿到 Token
# 2. 登录渠道
pnpm openclaw channels login --channel telegram
# （粘贴 token）

# 3. 重启 gateway 让渠道生效
pnpm openclaw gateway restart

# 4. 确认
pnpm openclaw channels status --probe
```

然后在 Telegram 搜到你的 bot，发 "hello"，看控制台有没有消息流进来。

## 2.8 建议你现在就建立的"工作桌面"

源码学习期间推荐打开这三样：

1. **编辑器窗口**：打开 `packages/` 下的子包
2. **终端 1**：常驻 `pnpm gateway:watch`（看日志）
3. **终端 2**：跑 `pnpm openclaw xxx` 子命令
4. **浏览器**：Control UI + Telegram 网页端

## 2.9 本章小结

- 从源码跑的五步：`clone → pnpm install → setup → ui:build → gateway:watch`
- 核心配置文件 `~/.openclaw/openclaw.json`：至少配好 `agent.model` + `providers.*.apiKey`
- 开发期常驻 `gateway:watch` + 一次 `openclaw doctor` 帮你排掉 80% 的基础坑
- 先接一个 Telegram bot，后续章节拆源码时你就有真实流量可以观察
