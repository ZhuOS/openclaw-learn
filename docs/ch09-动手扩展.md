# Ch09 · 动手：写一个自己的 Tool + Skill + Plugin

> 前 7 章全是拆解别人的代码，这一章轮到你动手。我们做一个完整的例子：**一个能查深圳天气的扩展**，三种形态都过一遍。

## 9.1 选题：为什么是天气

- 业务简单，不会被业务逻辑淹没
- 有真实外部 API 可以调，不是伪造
- 完美展示 Tool / Skill / Plugin 三者的协作

目标：让你的 OpenClaw Agent 能回答：

> "深圳今天天气怎么样？"

## 9.2 最简方案：只写一个 Skill（零代码）

**如果你只是想让 Agent 能查天气，连一行 TS 都不用写**——因为 OpenClaw 已经有 `web_fetch` 工具了。

### Step 1：建 Skill 目录

```bash
mkdir -p ~/.openclaw/skills/weather-shenzhen
cd ~/.openclaw/skills/weather-shenzhen
```

### Step 2：写 SKILL.md

```markdown
---
name: weather-shenzhen
description: |
  天气查询 Skill。当用户问及深圳、广州、珠三角等地的
  天气、气温、下雨、台风、空气质量时触发。
  触发词：天气、气温、下雨、台风、PM2.5、空气质量、出门、带伞
---

# 深圳天气查询

## 何时使用
当用户询问与天气相关的问题时（尤其是深圳及珠三角地区），使用本 Skill。

## 使用方法
1. 调用 `web_fetch` 工具访问：`https://wttr.in/Shenzhen?format=j1&lang=zh`
2. 从返回的 JSON 中提取：
   - `current_condition[0].temp_C` — 当前温度
   - `current_condition[0].weatherDesc[0].value` — 天气描述
   - `current_condition[0].humidity` — 湿度
   - `weather[0].hourly[]` — 未来逐小时
3. 用简洁中文回答，不要复述所有字段

## 回答格式
推荐的回答风格：

> 深圳现在 28°C，多云，湿度 75%。
> 下午有小雨，出门带伞。
> 今晚最低 24°C。

## 避坑
- wttr.in 偶尔不稳定，失败时提示用户"天气接口暂时不可用"，不要瞎编
- 不要把原始 JSON 贴给用户
```

### Step 3：重启 Gateway 让它被加载

```bash
openclaw gateway restart
```

### Step 4：测试

在任意渠道发："深圳今天天气怎么样？"

Agent 会：
1. 语义匹配到 `weather-shenzhen` skill
2. 读取 SKILL.md 里的指令
3. 调用 `web_fetch` 访问 wttr.in
4. 按你写的格式回复

**搞定了。零行 TypeScript。**

## 9.3 进阶：写一个专用 Tool

如果你觉得 Skill + `web_fetch` 太绕，或者想做更精细的控制（缓存、错误分类、限流），可以直接做成 Tool。

### Step 1：在 workspace 里建一个插件目录

```bash
mkdir -p ~/openclaw-weather-plugin/src
cd ~/openclaw-weather-plugin
```

### Step 2：package.json

```json
{
  "name": "openclaw-weather-plugin",
  "version": "0.1.0",
  "type": "module",
  "main": "./dist/index.js",
  "scripts": {
    "build": "tsc",
    "dev": "tsc --watch"
  },
  "openclaw": {
    "extensions": ["./src/index.ts"],

    "compat": {
      "pluginApi": ">=2026.3.24-beta.2",
      "minGatewayVersion": "2026.3.24-beta.2"

  },
  "dependencies": {
    "zod": "^3.23.0"
  },
  "devDependencies": {
    "typescript": "^5.5.0",

  }
}
```

### Step 3：tsconfig.json

```json

    "module": "ESNext",
    "moduleResolution": "bundler",

    "declaration": true
```

### Step 4：src/tool.ts

```ts
import { defineTool } from "@openclaw/plugin-sdk";
import { z } from "zod";

const CACHE = new Map<string, { at: number; data: WeatherResult }>();
const CACHE_TTL_MS = 10 * 60 * 1000; // 10 分钟

export interface WeatherResult {
  city: string;
  tempC: number;
  description: string;
  humidity: number;
  feelsLikeC: number;
  hourly: Array<{ time: string; tempC: number; desc: string }>;
}

export const getWeatherTool = defineTool({
  name: "get_weather",
  description:
    "获取指定城市的实时天气与未来数小时预报。支持全球大部分城市（基于 wttr.in）。" +
    "城市参数支持中文或英文（如 'Shenzhen'、'深圳'、'Shanghai'）。",
  schema: z.object({
    city: z.string().describe("城市名，中英文皆可"),
    lang: z.enum(["zh", "en"]).default("zh").describe("返回语言"),
  }),
  async execute({ city, lang }, ctx) {
    const cacheKey = `${city}:${lang}`;
    const hit = CACHE.get(cacheKey);
    if (hit && Date.now() - hit.at < CACHE_TTL_MS) {
      return { ok: true, cached: true, data: hit.data };
    }

    const url = `https://wttr.in/${encodeURIComponent(city)}?format=j1&lang=${lang}`;
    const controller = new AbortController();
    ctx.abortSignal?.addEventListener("abort", () => controller.abort());

    const res = await fetch(url, { signal: controller.signal });
    if (!res.ok) {
      return {
        ok: false,
        error: `wttr.in returned ${res.status}`,
      };
    }
    const raw = await res.json();

    const current = raw.current_condition?.[0];
    if (!current) {
      return { ok: false, error: "parse failed: no current_condition" };
    }

    const data: WeatherResult = {
      city,
      tempC: Number(current.temp_C),
      description: current.weatherDesc?.[0]?.value ?? "",
      humidity: Number(current.humidity),
      feelsLikeC: Number(current.FeelsLikeC),
      hourly: (raw.weather?.[0]?.hourly ?? []).slice(0, 6).map((h: any) => ({
        time: h.time,
        tempC: Number(h.tempC),
        desc: h.weatherDesc?.[0]?.value ?? "",
      })),
    };

    CACHE.set(cacheKey, { at: Date.now(), data });
    return { ok: true, cached: false, data };
  },
});
```

### Step 5：src/skill.ts

即使有了 Tool，**还是要配一个 Skill**——告诉 Agent "什么时候该调这个 tool"。

```ts
import type { SkillDefinition } from "@openclaw/plugin-sdk";

export const weatherSkill: SkillDefinition = {
  name: "weather-expert",
  description:
    "天气查询专家。当用户询问任何城市的天气、气温、湿度、未来几小时情况时使用。触发词：天气、气温、下雨、台风、湿度、带伞、出门。",
  content: `
# 天气查询专家

## 何时使用
当用户询问任何城市的天气（中外皆可），使用本 Skill 并调用 \`get_weather\` 工具。

## 使用方法
1. 从用户问题中提取城市（如"深圳天气"→"Shenzhen" 或 "深圳"）
2. 调用 \`get_weather\` 工具
3. 用 2-3 句自然中文回答，不要堆砌字段

## 回答示例
用户：深圳今天天气怎么样？
你调用 get_weather(city="Shenzhen")，拿到 {tempC: 28, description: "多云", humidity: 75, hourly: [...]}
你回答：深圳现在 28°C，多云，湿度 75%。未来几小时温度稳定，下午有短时阵雨，出门建议带伞。

## 避坑
- 不要把 hourly 全量贴出来，选 2-3 个关键时间点
- cached: true 时不要说"这是缓存数据"，直接回答即可
- 工具返回 ok: false 时，坦诚告知"天气接口暂时不可用"，不要瞎编
`,
};
```

### Step 6：src/index.ts

```ts
import { definePlugin } from "@openclaw/plugin-sdk";
import { getWeatherTool } from "./tool";
import { weatherSkill } from "./skill";

export default definePlugin({
  id: "weather",
  name: "OpenClaw Weather Plugin",
  version: "0.1.0",
  tools: [getWeatherTool],
  skills: [weatherSkill],
});
```

### Step 7：构建 + 本地安装

```bash
# 构建
npm install
npm run build

# 作为本地插件装入 OpenClaw
openclaw plugins install-local /absolute/path/to/openclaw-weather-plugin

# 重启
openclaw gateway restart

# 验证
openclaw plugins list
# 应看到 weather 插件
```

```bash

# 实际发布
clawhub package publish https://github.com/your-org/openclaw-weather-plugin@v0.1.0
```

**发布要求**：
- GitHub 账户 ≥ 1 周
- `package.json` 的 `openclaw.compat` 字段必须正确
- 自动构建成功

## 9.5 实战建议：什么时候用什么

| 需求 | 推荐方案 |
|------|---------|
| 改变 Agent 的"行为习惯"（例如"回答金融问题前先查数据"） | **Skill 就够了** |
| 有现成工具能做，只是需要指导 Agent | **Skill** |
| 需要自定义逻辑（缓存、错误分类、鉴权） | **Tool**（放进你的 Plugin） |
| 需要引入新的 LLM provider | **Plugin**（providers 字段） |
| 需要接入一个新的聊天平台 | **Plugin**（channels 字段） |
| 想让社区用上 | **发布到 ClawHub** |

## 9.6 调试技巧汇总

### 看 Agent 实际收到的 system prompt

```bash
# 临时环境变量
OPENCLAW_DEBUG_PROMPT=1 pnpm gateway:watch
```

会在日志里打印完整拼接后的 system prompt，能看到你的 SKILL.md 是怎么进去的。

### 看 tool 调用链

Control UI 里有"会话详情"面板，每次 tool_execution 都会显示：入参、出参、耗时。比翻 JSONL 快得多。

### 隔离测试你的 Tool

不要每次都在完整 Agent 里测。写个小脚本：

```ts
// test-weather.ts
import { getWeatherTool } from "./src/tool";

const result = await getWeatherTool.execute(
  { city: "Shenzhen", lang: "zh" },
  { abortSignal: undefined, workspaceDir: process.cwd(), sessionId: "test" } as any
);
console.log(JSON.stringify(result, null, 2));
```

`npx tsx test-weather.ts` 直接跑。

### AbortSignal 验证

故意让 tool 跑得慢，然后在 Control UI 里中断会话，看你的 fetch 有没有真的被 abort。**生产级 tool 必须支持 abort**。
